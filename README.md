Gemini said
This is one of the most confusing hurdles for everyone starting out! It feels like they should be the same thing, but in the tech world, Git and GitHub are two different tools that work together.

Think of it like this:

Git is the Engine (the software that actually tracks changes on your computer).

GitHub is the Garage (the place on the internet where you park your code so others can see it).

Even if you have a GitHub account and a repository online, your computer's "Command Prompt" doesn't know how to speak the language of "version control" until you install the Git engine.

📂 Course Notes: Git vs. GitHub
You should add this to your README later to remember the difference:

Git: Local software. Runs on your computer (Command Prompt/PowerShell).

GitHub: A website. Host for your Git repositories.

Git Bash: A special version of the command line that comes with the Git download. It’s "GitHub-friendly" and usually isn't blocked as easily as PowerShell!
